<h2><a id="user-content-analysis" class="anchor" aria-hidden="true" href="#analysis"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Analysis</h2>
<p>The original size of the dataset is 12GB which is too large for my preliminary analysis, I used the small dataset (128MB) to perform data exploration process.</p>
<p>After doing EDA, I created the below features for later modeling part.</p>
<ul>
<li>artist: the number of artists</li>
<li>gender: 0 or 1</li>
<li>length: the total length</li>
<li>level: 0 or 1</li>
<li>page: the number of Thumbs Up / Thumbs Down</li>
<li>song: the number of songs</li>
</ul>
<p>I have used three machine learning models:</p>
<ul>
<li>Logistic Regression</li>
<li>Random Forest classifier</li>
<li>GBT classifier</li>
</ul>
<p>As a result of the imbalanced dataset (<code>Churn</code> users are extremely few), Logistic Regression predicted zeros (<code>Not Churn</code>) for all the users and gives me F1 score 0.732. Even</p>
<p>I decided to use this score as a baseline, and better scores than this baseline are necessary for further modeling.</p>
<p>Testing three machine learning models and Random Forest gives me the best score which is 0.738. According to the below feature importance provided by the Random Forest model, <code>Thumbs Up</code> and <code>Thumbs Down</code> seem to be important while the level of the users do not really matter.</p>
